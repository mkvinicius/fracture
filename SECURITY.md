# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.1.x   | ✅ Yes    |
| 1.0.x   | ✅ Yes    |
| < 1.0   | ❌ No     |

## Reporting a Vulnerability

If you discover a security vulnerability in FRACTURE, please **do not** open a public GitHub issue.

Instead, report it privately by opening a [GitHub Security Advisory](https://github.com/mkvinicius/fracture/security/advisories/new).

We will acknowledge your report within 48 hours and aim to release a fix within 14 days for critical issues.

## Scope

- **In scope:** Remote code execution, data exfiltration, authentication bypass, injection attacks
- **Out of scope:** Issues requiring physical access, social engineering, or denial-of-service on self-hosted instances
